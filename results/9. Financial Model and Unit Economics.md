## Executive Summary

The OpenHuman unit economics model is **viable as a free-tier-as-standard play, but only inside a narrow operating envelope defined by one variable: cache hit rate.** At the Base Case (70% cache hit, ~$0.0028 blended cost per active session), the architecture sustains a 100M MAU free tier on roughly $2.8M/month of variable infrastructure spend — well within enterprise-SDK gross margin coverage assuming even modest B2B monetization (~150 paying partners at ARPU $4K/mo by Year 3). At 50% cache hit (Bear), cost/session balloons to $0.0058 — **above the $0.005 free-tier sustainability ceiling** — and the standard strategy collapses into a capital-burn problem. At 90% (Bull), cost/session falls to $0.0011 and the cost MOAT becomes structural and durable.

**The single most critical assumption to de-risk in the next 90 days is the achievable cache hit rate at MVP launch.** Everything downstream — free-tier viability, partner CAC payback, EBITDA crossover timing, and the entire competitive moat thesis — pivots on whether the pre-compute pipeline (OpenHuman-Brain context tree caching + Cloudflare edge cache + phoneme-level reuse) can credibly hit 50%+ on cold launch and 70%+ within 6 months. We recommend a pre-MVP cache simulation against a representative query distribution (chitchat + top-3 domain trees) before Phase 4 protocol freeze. The secondary risk to monitor is GPU spot pricing volatility on Hetzner — a 40% adverse move pushes Bear blended cost to $0.0078, breaking the model regardless of cache performance.

**Funding implication:** A $6M Seed funds Months 1–16 (Phase 1 through closed beta) with ~3 months buffer at Base burn. A Series A of $18–25M is required at Month 14–16, gated on demonstrated cache hit ≥60% and ≥3 enterprise SDK LOIs.

# DELIVERABLE 1 — UNIT ECONOMICS DASHBOARD

## 1.1 Primary unit definition

**Active session** = one unique user-initiated WebSocket connection lasting ≥30 seconds with at least one full request/response turn (ASR → Brain → Voice + Motion → render). This is the billable unit for B2B SDK pricing and the cost-attribution unit internally.

**Why "session" not "user":** OpenHuman is embedded infrastructure. A single end-user may generate 0 sessions/month (embedded but unused) or 200 sessions/month (heavy kiosk user). Sessions normalize across embedding contexts (Zoom, Discord, kiosk, mobile) where "user" definitions diverge.

## 1.2 Cost per active session — by traffic type

**Cost stack inputs (Base Case, derived in §4 Assumption Register):**

| Cost component                    | Unit cost           | Driver                                                                      |
| --------------------------------- | ------------------- | --------------------------------------------------------------------------- |
| GPU inference (cache miss)        | $0.0042 / session   | A100-equivalent @ $1.80/GPU-hr, 350ms avg compute, batched 2x               |
| CPU compute (cache hit serve)     | $0.00018 / session  | Hetzner AX102 EPYC, 1M concurrent WS/box @ $290/mo                          |
| Cloudflare egress (CDN-cached)    | $0.000045 / session | Free tier + CF Pro overage at $0.04/GB; 90 sec session × 100 kbps = 1.13 MB |
| Cloudflare egress (origin-served) | $0.00012 / session  | Hetzner egress $1/TB, but routed via CF still incurs CF cost on miss        |
| R2 storage (amortized)            | $0.000008 / session | $0.015/GB/mo R2; pre-computed asset cache amortized over reads              |
| Postgres + memory ops             | $0.000022 / session | RDS-equivalent, ~3 reads + 1 write per session                              |
| Protocol/anti-crawl overhead      | $0.000005 / session | Marginal CPU cycles for delta encoding + obfuscation                        |

### Per-session cost by traffic type

| Traffic type                       | Components                                        | Cost / session |
| ---------------------------------- | ------------------------------------------------- | -------------- |
| **Cache hit (CPU-only, CF edge)**  | CPU + CF cached egress + R2 + PG + Protocol       | **$0.00026**   |
| **Cache miss (full GPU pipeline)** | GPU + CPU + CF origin egress + R2 + PG + Protocol | **$0.00440**   |

### Blended cost / session (Base Case, 70% cache hit)

```
Blended = (0.70 × $0.00026) + (0.30 × $0.00440)
        = $0.000182 + $0.001320
        = $0.00150 / session

Including ~85% cost-allocation overhead (idle GPU capacity,
training amortization, observability, cross-AZ replication):

Effective blended = $0.00150 × 1.85 = ~$0.00278 / session ≈ $0.0028
```

> 🟢 **Result: $0.0028/session is below the $0.005 free-tier ceiling — Base Case viable.** 30% margin against the red-flag threshold gives operating headroom for traffic spikes and adverse GPU pricing within ±20%.

## 1.3 Cost per embedded deployment (B2B SDK partner)

Modeled as cost to serve one third-party app's monthly active users.

**Sessions per MAU/month assumption:** 12 (one session every 2.5 days) — typical mid-engagement consumer app (kiosk skews higher at 60+, enterprise tools lower at 4–6).

| Partner profile            | MAU served | Sessions/mo | Cost/MAU/mo (chitchat-heavy) | Cost/MAU/mo (domain-specialized) |
| -------------------------- | ---------- | ----------- | ---------------------------- | -------------------------------- |
| Small SaaS embed           | 10K        | 120K        | $0.034                       | $0.061                           |
| Mid-market app             | 250K       | 3M          | $0.034                       | $0.061                           |
| Large platform (Zoom-tier) | 10M        | 120M        | $0.030 (volume CF discount)  | $0.054                           |

**Why domain-specialized costs ~80% more:** medical/legal/scientific context trees have lower cache hit rates (50–55% vs. 75–80% for chitchat) because queries are more specific and less repetitive. They also call larger LLMs in the OpenHuman-Brain dispatcher.

## 1.4 LTV of enterprise embedding partner

**SDK pricing tiers (modeled):**

| Tier               | Monthly fee | MAU cap                | Target segment                                 |
| ------------------ | ----------- | ---------------------- | ---------------------------------------------- |
| Free / Open Source | $0          | <10K                   | Indie devs, prototypes — funnel for paid tiers |
| Startup            | $499/mo     | 100K                   | YC-stage and bootstrapped                      |
| Growth             | $4,000/mo   | 2M                     | Series B+ SaaS, mid-market platforms           |
| Enterprise         | $25,000+/mo | Unlimited + custom SLA | Zoom/Discord-tier, large enterprises           |

**Blended ARPU across paid partners (weighted by expected mix):**

- 60% Startup ($499) + 30% Growth ($4,000) + 10% Enterprise ($25,000)
- **Blended ARPU = $4,000/month per paying partner**

**Gross margin per partner profile (Base Case):**

| Tier        | Monthly revenue | Infra cost (cost/MAU × MAU at cap) | Gross margin |
| ----------- | --------------- | ---------------------------------- | ------------ |
| Startup     | $499            | ~$3.40 (100K MAU × $0.034)         | **99.3%**    |
| Growth      | $4,000          | ~$68 (2M × $0.034)                 | **98.3%**    |
| Enterprise  | $25,000         | ~$300 (10M × $0.030)               | **98.8%**    |
| **Blended** | **$4,000**      | **~$50 (avg)**                     | **~98.7%**   |

> Note: gross margin is extremely high because the free tier absorbs most infrastructure cost; paying partners are subsidized only at the marginal session cost. This is the same dynamic as Twilio (~50% GM) and Mapbox (~70% GM) but more favorable because the cache layer is structural, not per-call.
>
> **Conservative gross margin used in LTV calc: 75%** (haircut for support, customer success, partner-specific dev work, free-tier subsidy attribution).

**Monthly partner churn rate assumption:** 2.5% (B2B infrastructure benchmark — Twilio ~1.5%, Mapbox ~2%, riskier given pre-revenue + competitive landscape)

```
LTV = (ARPU × Gross Margin %) ÷ Monthly Churn
    = ($4,000 × 0.75) ÷ 0.025
    = $3,000 ÷ 0.025
    = $120,000 per enterprise partner
```

## 1.5 LTV:CAC and payback period

**CAC assumptions for enterprise SDK channel:**

| Cost item                                             | Per acquired partner |
| ----------------------------------------------------- | -------------------- |
| Outbound SDR + AE blended                             | $14,000              |
| Solutions engineering (integration support pre-close) | $8,000               |
| Marketing attribution (events, content)               | $3,000               |
| **Total CAC**                                         | **$25,000**          |

```
LTV:CAC = $120,000 ÷ $25,000 = 4.8 : 1   ✅ above 3:1 threshold

Payback = CAC ÷ (ARPU × GM%)
        = $25,000 ÷ ($4,000 × 0.75)
        = $25,000 ÷ $3,000
        = 8.3 months   ✅ well below 18-month threshold
```

> 🟢 **Both metrics pass benchmark.** However, both are sensitive to ARPU mix — if Enterprise tier conversion lags and the mix skews to Startup ($499), blended ARPU drops to ~$1,200, LTV falls to ~$36K, and LTV:CAC compresses to 1.4:1. **Mix risk is the dominant LTV sensitivity, not churn.**

## 1.6 Contribution margin per session

For a **paid B2B-attributed session** (revenue allocated proportionally per session served to that partner's MAU):

```
Revenue per session (attributed):  $0.0044   ($4,000/mo ÷ ~900K avg sessions/Growth partner)
Variable cost per session:        -$0.0028
─────────────────────────────────────────────
Contribution margin per session:   $0.0016   (36.4%)
```

For **free-tier sessions:** contribution margin = -$0.0028 (full subsidy). Free-tier subsidy is funded by paid-partner contribution margin and platform-wide network effect value.

**Cross-subsidy ratio:** at 70% cache hit and current pricing, **1 paid Growth-tier MAU funds ~118 free-tier MAU**. This is the math that makes the 100M free-tier strategy viable: only ~850K paid MAU equivalent is needed to break even on infra cost at full scale.

## 1.7 Cache hit rate sensitivity (most critical lever)

| Cache hit rate | Cache hit cost share | Cache miss cost share | **Blended cost/session** | vs. Base | Free-tier viable? |
| -------------- | -------------------- | --------------------- | ------------------------ | -------- | ----------------- |
| 50%            | $0.00013             | $0.00220              | **$0.00432** (×1.85 OH)  | +54%     | ⚠️ Borderline     |
| 60%            | $0.00016             | $0.00176              | **$0.00355** (×1.85 OH)  | +27%     | 🟡 Tight          |
| **70% (Base)** | $0.00018             | $0.00132              | **$0.00278** (×1.85 OH)  | —        | 🟢 Viable         |
| 80%            | $0.00021             | $0.00088              | **$0.00202** (×1.85 OH)  | -27%     | 🟢 Strong         |
| 90%            | $0.00023             | $0.00044              | **$0.00125** (×1.85 OH)  | -55%     | 🟢 Cost MOAT      |

> 🔴 **At 50% cache hit rate, blended cost is $0.0043 — uncomfortably close to the $0.005 free-tier sustainability ceiling.** Under any GPU pricing pressure (+15% adverse), the 50% scenario crosses the threshold and the free-tier strategy becomes capital-funded rather than self-sustaining. **This is the single most important number in the model.**

## 2.5 Break-even analysis

### Session-level break-even

The cache hit rate at which blended cost reaches the $0.001 stretch target (10× cushion below $0.005 ceiling):

```
Solve: (HitRate × $0.00026 × 1.85) + ((1-HitRate) × $0.00440 × 1.85) = $0.00100

→ HitRate = 91.2%
```

> **At 91% cache hit, infra cost per session reaches $0.001 — the Mapbox/Twilio-class cost structure required for true free-tier dominance.** This is achievable in Year 3 per Phase 7 targets but is NOT the MVP target.

### Company-level break-even

| Milestone                    | When                 | MAU  | Paying partners | Revenue    | Variable infra cost               | Contribution covers fixed? |
| ---------------------------- | -------------------- | ---- | --------------- | ---------- | --------------------------------- | -------------------------- |

| Contribution margin positive | Q1Y3                 | 1.5M | 75              | $720K/qtr  | $590K/qtr (assuming better cache) | 🟡 Marginal                |
| EBITDA break-even            | **Q3Y4 (projected)** | ~30M | ~400            | ~$5.6M/qtr | ~$5.4M/qtr                        | 🟢 Yes                     |

> **EBITDA break-even is ~Month 33 (Q3Y4) under Base Case** — typical for infrastructure standards (Twilio took 9 years, Mapbox ~7). Not alarming for the category, but requires a deliberate Series B and likely Series C path.

## 5.2 Red flag triggers and corrective actions

### 🔴 Red Flag 1: Blended cost/session exceeds $0.005

| Threshold      | Current Base | Bear Case | Trigger               |
| -------------- | ------------ | --------- | --------------------- |
| $0.005/session | $0.0028      | $0.0060   | Bear scenario crosses |

**Downstream impact:** Free tier becomes capital-funded rather than self-sustaining. At 100M MAU × 12 sessions/mo × $0.005 = $6M/mo subsidy required vs. $3.4M at Base. Standard adoption stalls because partners cannot embed without economic risk.

**Corrective actions, mapped to modules:**

1. **OpenHuman-Cache:** increase TTL on chitchat tree from 24h → 7d; expand pre-compute corpus from top-1K to top-10K queries
2. **OpenHuman-Brain:** route low-confidence cache misses to a smaller distilled LLM before escalating to full model
3. **OpenHuman-Motion / Voice:** push more aggressive INT4 quantization; accept marginal MOS degradation
4. **OpenHuman-Protocol:** add session-level prefetching of common follow-up turns

### 🔴 Red Flag 2: Cache hit rate at MVP launch <50%

**Threshold:** <50% measured cache hit rate in closed beta (Phase 6).

**Downstream impact:** Invalidates the entire cost MOAT thesis. Forces a re-architecture decision: either (a) accept cache-per-user model with lower hit rates, or (b) delay launch to expand pre-compute corpus.

**Corrective actions:**

1. **OpenHuman-Brain:** expand pre-compute pipeline coverage — pre-render top 80% queries per context tree before launch; add chitchat warmup batch on cold deployment
2. **OpenHuman-Cache:** increase chitchat tree TTL to 30 days; add semantic similarity matching (not just exact prompt hash) for near-duplicate queries
3. **OpenHuman-Crawler:** review topic diversity — narrow to higher-frequency conversational patterns; deprioritize long-tail content for MVP

### 🔴 Red Flag 3: GPU inference first-byte latency >500ms

**Threshold:** p95 first-byte latency exceeds 500ms.

**Downstream impact:** Lip-sync UX breaks. User perceives latency as "broken" rather than "slow." Catastrophic for product-market fit; partners refuse to embed.

**Corrective actions:**

1. **OpenHuman-Motion / Voice:** further distillation; consider speculative decoding for motion curves
2. **OpenHuman-Backend:** move GPU workers closer to WebSocket terminators; reduce gRPC hop overhead
3. **OpenHuman-Protocol:** stream first chunk earlier — emit motion delta on partial mel-spec rather than full sentence

### 🔴 Red Flag 4: Enterprise SDK CAC exceeds $35K with LTV:CAC <3:1

**Threshold:** CAC creep above $35K (40% above modeled $25K) AND LTV:CAC below 3:1.

**Downstream impact:** Payback exceeds 12 months. Series B at risk because GTM efficiency falls outside category benchmarks. Burns through Series A faster than modeled.

**Corrective actions:**

1. Shift CAC mix from outbound to product-led growth via the free SDK tier (per Mapbox playbook)
2. Increase free→paid conversion via usage limits at the Free tier
3. Develop case-study content from initial 5 lighthouse partners to compress sales cycle

### 🔴 Red Flag 5: Cloudflare egress grows faster than session volume

**Threshold:** Egress GB/month grows >1.2× the rate of session volume month-over-month.

**Downstream impact:** Delta encoding is not delivering modeled bandwidth savings. At 100M MAU, even a 2× bandwidth overrun means $1M+/mo unplanned CF egress.

**Corrective actions:**

1. **OpenHuman-Protocol:** audit delta encoding effectiveness; implement reset frame frequency tuning; consider 4-bit deltas for low-motion frames
2. **OpenHuman Render Engine:** push more interpolation client-side; reduce reset frame frequency
3. **Cloudflare:** negotiate committed-spend egress discount earlier than planned

### 🔴 Red Flag 6: Data acquisition cost exceeds $40 per hour of extracted curves

**Threshold:** OpenHuman-Animator GPU batch cost per hour of validated 135-curve output exceeds $40 (vs. modeled ~$31 = $1.25/GPU-hr ÷ 4 hrs throughput, with overhead).

**Downstream impact:** 10K-hour dataset target costs $400K+ vs. modeled $310K. Compounds because dataset size is the data MOAT — undershooting compromises the entire competitive thesis.

**Corrective actions:**

1. **OpenHuman-Animator:** improve TrueDepth synthesis plugin throughput; profile UE5 batch pipeline for I/O bottlenecks
2. **OpenHuman-Crawler:** pre-filter video quality more aggressively to reduce wasted extraction on low-yield content
3. Negotiate dedicated Hetzner GPU contract (vs. on-demand) at scale-out
